# Task Manager с LLM-функциями

Приложение для управления задачами с интеграцией LLM-функций, построенное на NextJS 16.

## 1. Краткое описание проекта и реализованных функций

### Управление задачами (CRUD)
- Создание, просмотр, редактирование и удаление задач
- Атрибуты: название, описание, приоритет, статус, категория, срок выполнения

### Фильтрация и поиск
- Фильтрация по статусу и приоритету
- Полнотекстовый поиск по названию и описанию
- Комбинируемые фильтры
- Серверная логика фильтрации (SQL WHERE)

### LLM-функции
- **Умная категоризация** — предлагает категорию на основе названия и описания
- **Предложение приоритета** — анализирует срок и содержание задачи
- **Декомпозиция задач** — разбивает сложную задачу на подзадачи
- **Сводка нагрузки** — генерирует естественно-языковую сводку по задачам

### Продвинутые LLM-функции
- **Streaming** — потоковая передача ответов для сводки и декомпозиции
- **Кэширование** — 5-минутный TTL для идентичных запросов
- **Fallback-механизмы** — graceful degradation при недоступности LLM
- **Few-shot промпты** — примеры в системных промптах для всех функций

### UX/UI
- Цветовая схема по теории цвета (комплементарные цвета HSL)
- Адаптивный дизайн
- Skeleton-экраны и прогресс-бары
- Плавные анимации и переходы

## 2. Пошаговые инструкции настройки среды

### Требования
- Node.js 18+

### Установка

```bash
npm install
```

### Переменные окружения

Создайте файл `.env.local` (опционально):

```env
GIGACHAT_CREDENTIALS=your_credentials
```

По умолчанию используются встроенные демо-credentials.

## 4. Архитектурные решения

### NextJS App Router
- Server Components для рендеринга
- API Routes для бэкенда
- Turbopack для быстрой сборки

### База данных
- SQLite через better-sqlite3
- Хранится в файле tasks.db

### UI
- Material UI (MUI) с кастомной темой
- HSL-цвета для гибкости настройки
- Комплементарная цветовая схема (primary: 221°, secondary: 175°, accent: 280°)

### LLM Integration
- GigaChat API (Sber)
- Server-side streaming (SSE)
- In-memory кэширование с TTL
- Fallback-логика при ошибках

### Промпт-инжиниринг
- Системные роли для каждого действия
- Few-shot примеры в промптах
- JSON-формат вывода с валидацией
- Обработка некорректных ответов

## 5. Известные ограничения

1. Нет аутентификации
2. Нет пагинации
3. LLM требует интернет
4. БД better-sqlite3 не подходит для serverless-платформ
5. Кэш in-memory (сбрасывается при перезапуске)
6. Меньшая приватность облачной ИИ по сравнению с локальными моделями
